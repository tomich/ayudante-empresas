*Realizar pings o traceroutes a sitios externos para comprobar perdida de paquetes

la forma mas facil de acceder es con el Macro de CAT o con el Script en VBS: Scripts Tom

En maquinas con perfil CAT o PYME o Empresa se puede simplemente acceder a "Acceso directo a Router - Passports v1.6-Tecnicos" que deberia encontrarse en el escritorio.

En maquinas sin este perfil, realizar lo siguiente:
Abrir consola (inicio-ejecutar-cmd)
escribir el siguiente comando:
telnet 10.0.24.13

Los datos de acceso son:
Usuario: soporte
Clave: sop3767
Una vez aqui dentro, podemos realizar ping a todos los CPE/IAD que deseemos. Y acceder por telnet a varios equipos de la misma forma que accedimos a este, es decir, desde esta nueva consola escribimos el comando:

telnet IPDELROUTER
usuario
clave

**¿Para que nos sirve el ping?**
-inicialmente para saber si ese destino existe(si el destino tiene IP o si la IP existe/responde)
-Para saber el IP de un dominio (un ping a www.google.com nos indicara el IP de la pagina por ejemplo)
-Para saber el tiempo de respuesta en milisegundos que tiene ese destino

**¿Para que nos sirve acceder por telnet a un CPE/IAD?**
-Para Verificar si el equipo tal vez esta colgado de forma que sincroniza, conecta pero no tiene acceso de ningun tipo.
-Para acceder a un equipo que no permite el acceso web o que tiene el acceso web colgado (por ejemplo en un CPE Comtrend 536+ podemos hacer las mismas tareas que por web)
-Para verificar si el equipo puede "navegar", para hacer esto realizamos telnet al equipo y una vez dentro del destino, hacemos un ping, por ejemplo a Google. Un ejemplo sencillo de este caso seria que el usuario no puede navegar y google.com no responde ping, pero en el CPE si responde, entonces sabemos que no es un error del servicio HASTA el CPE.
-Al punto anterior se suma que algunos CPE's/IAD pueden presentar bloqueos donde sincronizan, navegan, pero tienen navegacion nula o no pueden ingresar a alguna pagina especifica.
-En equipos que tengan un firmware basado en sistemas UNIX(casi todos) podemos acceder al Shell de comandos con el comando "sh" (una vez dentro del equipo) y realizar tareas adicionales, aunque en la mayoria de los quipos jazztel los comandos son los mismos que apenas ingresamos por telnet.
Hay ocaciones donde (si se saben los comandos de unix) podemos verificar si un puerto esta abierto, comprobar la tabla nat en iptables, reiniciar puertos, verificar el % de uso del CPU del equipo ,la memoria RAM libre, un prceso colgado, la transferencia desde alguna interfaz, etc... que nos puede servir para diagnosticar algun fallo especifico.
